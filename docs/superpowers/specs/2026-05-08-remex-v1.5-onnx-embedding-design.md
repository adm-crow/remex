# Remex v1.5.0 ‚Äî ONNX Embedding Backend Design

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the SentenceTransformers (PyTorch) embedding backend with ONNX Runtime, eliminating the ~2 GB PyTorch dependency, bypassing corporate HuggingFace CDN blocks, and bundling the default model in the installer for offline-first first use.

**Architecture:** A new `remex/core/embedding.py` module exposes a single factory `get_embedding_function(model_name)` that routes the default model through ChromaDB's built-in `ONNXMiniLM_L6_V2` (downloads from Chroma's S3, not HuggingFace) and all other models through a custom FastEmbed wrapper. `pipeline.py` calls this factory instead of the current `_RemexEmbeddingFunction` subclass.

**Tech Stack:** Tauri v2 (Rust), React 19 + TypeScript + Vite, Zustand, Vitest + Testing Library, Python 3.13 FastAPI sidecar, ChromaDB 1.5.x, FastEmbed (ONNX Runtime).

**Breaking change:** Embedding vectors are incompatible with v1.4.x collections. All existing collections must be deleted and re-ingested after upgrading.

## 1. Embedding Backend (Python)

### New file: `remex/core/embedding.py`

Single responsibility: own the embedding function factory, cache, and lock.

```python
# Public surface
def get_embedding_function(model_name: str) -> EmbeddingFunction: ...
def warmup(model_name: str) -> None: ...  # used by warmup_collections
```

**Routing logic:**
- `model_name == "all-MiniLM-L6-v2"` ‚Üí `ONNXMiniLM_L6_V2()` from `chromadb.utils.embedding_functions`
  - Downloads from `https://chroma-onnx-models.s3.amazonaws.com/all-MiniLM-L6-v2/onnx.tar.gz`
  - No HuggingFace involved ‚Äî works on Capgemini and similar corporate networks
- Any other model name ‚Üí `_FastEmbedEmbeddingFunction(model_name)`
  - Thin wrapper around `fastembed.TextEmbedding`
  - Downloads from HuggingFace Hub on first use

**`_FastEmbedEmbeddingFunction`:**
```python
class _FastEmbedEmbeddingFunction(EmbeddingFunction):
    def __init__(self, model_name: str) -> None:
        self._model = TextEmbedding(model_name=model_name)

    def __call__(self, input: Documents) -> Embeddings:
        return [list(v) for v in self._model.embed(input)]
```

**Cache:** `_EF_CACHE: dict[str, EmbeddingFunction]` + `_EF_LOCK: threading.Lock()` move from `pipeline.py` into `embedding.py`. Cache key is the model name string.

### Changes to `remex/core/pipeline.py`

- Delete `_RemexEmbeddingFunction`, `_EF_CACHE`, `_EF_LOCK`
- Replace `_get_ef(model_name)` with `from .embedding import get_embedding_function`
- All call sites of `_get_ef(...)` become `get_embedding_function(...)`
- No other logic changes

### Dependencies (`pyproject.toml`)

- **Remove:** `sentence-transformers`
- **Add:** `fastembed` (brings `onnxruntime` transitively ‚Äî required by both backends)
- PyTorch is no longer a transitive dependency ‚Üí venv shrinks from ~2 GB to ~300 MB

## 2. Default Model Bundling

### Goal

First ingest on any machine ‚Äî including fully offline or heavily firewalled ‚Äî requires zero network calls for the default model.

### Build-time (CI)

In `.github/workflows/release.yml`, before `tauri build`, add a step:

```yaml
- name: Download default ONNX model
  run: |
    curl -L https://chroma-onnx-models.s3.amazonaws.com/all-MiniLM-L6-v2/onnx.tar.gz \
      -o onnx.tar.gz
    mkdir -p studio/src-tauri/resources/onnx/all-MiniLM-L6-v2
    tar -xzf onnx.tar.gz -C studio/src-tauri/resources/onnx/all-MiniLM-L6-v2
```

### Tauri bundle config (`tauri.conf.json`)

```json
"bundle": {
  "resources": {
    "resources/onnx/all-MiniLM-L6-v2/*": "onnx/all-MiniLM-L6-v2/"
  }
}
```

### Runtime (Rust ‚Äî `lib.rs`)

In `do_spawn()`, after setting `HF_HUB_DISABLE_SYMLINKS_WARNING`, set:

```rust
let bundled_onnx = resource_dir.join("onnx").join("all-MiniLM-L6-v2");
if bundled_onnx.exists() {
    cmd.env("REMEX_BUNDLED_ONNX_PATH", &bundled_onnx);
}
```

### Runtime (Python ‚Äî remex CLI `serve` startup)

In the `serve` command entry point, before starting uvicorn:

```python
def _seed_bundled_model() -> None:
    bundled = os.environ.get("REMEX_BUNDLED_ONNX_PATH")
    if not bundled:
        return
    dest = Path.home() / ".cache" / "chroma" / "onnx_models" / "all-MiniLM-L6-v2" / "onnx"
    if dest.exists():
        return
    dest.mkdir(parents=True, exist_ok=True)
    for f in Path(bundled).iterdir():
        shutil.copy2(f, dest / f.name)
```

## 3. Proxy Support

In `lib.rs` `do_spawn()`, after the `HF_HUB_DISABLE_SYMLINKS_WARNING` env set:

```rust
for var in &[
    "HTTP_PROXY", "HTTPS_PROXY", "NO_PROXY",
    "http_proxy", "https_proxy", "no_proxy",
] {
    if let Ok(val) = std::env::var(var) {
        cmd.env(var, val);
    }
}
```

No UI changes. `requests`, `httpx`, and `huggingface_hub` all read these env vars automatically. Silent passthrough ‚Äî if nothing is set in the user's session, nothing changes.
